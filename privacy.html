Política de Privacidad y Manejo de Datos

AL UTILIZAR RADAR-SD (SISTEMA VANGUARD), USTED ACEPTA EXPRESAMENTE EL PROCESAMIENTO DE SUS DATOS DE UBICACIÓN Y EL USO DE SENSORES DE HARDWARE BAJO LOS TÉRMINOS DESCRITOS EN ESTE DOCUMENTO.

Radar-SD no es una aplicación convencional basada en la nube. Es una red de vigilancia urbana descentralizada y P2P (Peer-to-Peer). Esta arquitectura redefine la privacidad, eliminando la necesidad de servidores centrales y otorgando al usuario el control absoluto de su información.

1. Identidad Criptográfica y Anonimato

A diferencia de otras plataformas, Radar-SD no solicita nombres, correos electrónicos, números de teléfono ni perfiles de redes sociales. Tu identidad en la red es puramente criptográfica.

  • Generación de Llaves: Al iniciar, el sistema genera un par de llaves asimétricas (ECDSA) utilizando el hardware seguro de tu dispositivo (Android KeyStore / StrongBox).
  • Identificador de Nodo (NODO_ID): Tu identidad pública es un hash irreversible de tu llave pública. Este ID permite que otros nodos verifiquen la autenticidad de tus reportes sin conocer tu identidad real.
  • Firma Digital: Cada acción que realizas (reportar o validar) es firmada digitalmente. Solo tú, a través de tu llave privada protegida por hardware, puedes autorizar estas transmisiones.

2. Uso y Tratamiento de la Ubicación

La ubicación es la columna vertebral de Radar-SD. Sin ella, la red mesh no puede operar. Procesamos la ubicación de dos formas:

A. Posicionamiento en el Mapa (GPS)

Utilizamos el GPS para fijar con precisión milimétrica la ubicación de las anomalías urbanas (baches, fallas eléctricas, etc.). Estos datos se guardan en tu base de datos local y se comparten con otros nodos solo cuando tú decides "Transmitir".

B. Descubrimiento de Proximidad (Nearby)

Para versiones de Android anteriores a la 12, se requiere el permiso de ubicación para permitir que el protocolo Google Nearby Connections identifique otros teléfonos cercanos y construya la red mesh de sincronización.

3. Uso de la Cámara y Multimedia

El acceso a la cámara se solicita exclusivamente para la captura de evidencia técnica. Radar-SD utiliza un motor de IA Local (TensorFlow Lite) para analizar la imagen en tiempo real y asignar un nivel de confianza al reporte. Las imágenes se almacenan en el directorio privado de la aplicación y se transfieren a otros nodos durante la sincronización táctica.

4. Almacenamiento y Derechos del Usuario

Toda la información que recibes de otros ciudadanos se almacena en una base de datos SQLite (Room) en tu dispositivo. Radar-SD no posee una "nube" donde se guarden estos datos.

  • Derecho al Olvido: Puedes eliminar cualquier reporte que hayas creado desde la pantalla de detalles. Esto enviará una señal de purga a la red mesh para informar que el dato debe ser eliminado.
  • Limpieza de Caché: En la sección de ajustes, puedes eliminar la caché de mapas (OpenStreetMap) para liberar espacio de almacenamiento.
  • Eliminación Total: Al desinstalar la aplicación o limpiar los datos desde los ajustes del sistema Android, toda tu base de datos y tu identidad criptográfica se borrarán permanentemente. No existe forma de recuperar una identidad borrada.

5. Seguridad en la Transmisión (P2P)

Aunque no hay un servidor central, los datos están protegidos durante el tránsito:

Protocolo Vanguard: Los datos se transfieren mediante túneles cifrados de Google Nearby. Además, cada reporte incluye una firma ECDSA y una prueba de trabajo (Proof of Work) para prevenir ataques de spam y asegurar la integridad de la información recibida de otros nodos.

6. Servicios de Terceros

Para ofrecer una experiencia operativa completa, integramos:

  • Google Play Services: Utilizado para la infraestructura de red Nearby Connections.
  • OpenStreetMap (OSM): Proveedor de la cartografía táctica. Las peticiones de mapas pueden exponer tu dirección IP al servidor de tiles de OSM, pero no tu identidad de Radar-SD.

7. Actualizaciones

Esta política puede actualizarse para reflejar mejoras en los protocolos de red. Se recomienda revisar el "Centro de Utilidades" periódicamente.
